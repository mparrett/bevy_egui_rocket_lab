# Repo Review — Feb 2026

Picking this project back up after leaving it messy in Aug 2024. Here's what was found.

## Untracked files — three categories

### Junk / safe to delete

- `.DS_Store`, `assets/.DS_Store`, `assets/textures/.DS_Store` — macOS cruft
- `_temp/` (2.2MB) — single source PNG for grass texture, already converted to ktx2
- `terst/` (3.4GB!) — full clone of the project with its own `.git` and `target/` dir. Likely a test copy. This is the big one
- `out/` (36MB) — wasm-bindgen build output, regenerated by `just release-wasm`
- `bevy-repo/` (129MB) — cloned bevy source, probably for reference/debugging
- `.cargo/`, `.vscode/` — local editor/toolchain config

### Asset experiments (uncommitted, was deciding what to keep)

- `assets/Ryfjallet_cubemap.psd` (1.9MB) — Photoshop source for the skybox cubemap
- `assets/textures/Cubemap-export-test/` — test exports from the PSD
- `assets/textures/Ryfjallet_cubemap_gray*` — grayscale cubemap variants (experimenting with sky look)
- `assets/textures/GroundGrassGreen002_*` — multiple resolutions (1024, 2048) of grass bump/color/metallic roughness textures, plus ktx2 conversions
- `assets/textures/green+grass-*` — alternative grass textures in ktx2/etc2 formats
- `assets/environment_maps/` (11MB) — pisa + cubes reflection probe maps
- `assets/uneven_ground_mesh.glb` — 3D mesh, likely for the terrain work

### Useful stuff to commit

- `scripts/` — texture/audio conversion scripts with a README pointing to KTX-Software. These support the asset pipeline and belong in the repo

## Branches — a tangled mess

- **`outstanding-changes`** — 3 commits ahead of main with real cleanup work (clippy fixes, removing unused params, simplifying queries). **Unmerged and has value.**
- **`MERGED-*` branches (4)** — Despite the prefix, none are actually merged into main. They look like intermediate staging branches from when the history was being untangled. Probably safe to delete since their content was cherry-picked/redone in later branches.
- **`mattmp`** — Long-running dev branch with ~50+ WIP commits. The meaningful changes appear to have been extracted into `outstanding-changes` and the PR branches already.
- **`mattmp-explode`** and **`main-for-explode`** — Attempts to reorganize/cherry-pick. Likely obsolete.

## Recommended cleanup

1. **Delete immediately** (no commit needed):
   - `terst/` — 3.4GB test clone, reclaim disk space
   - `_temp/` — source PNG already processed
   - `out/` — build artifact, regenerated on demand
   - `.DS_Store` files

2. **Add to `.gitignore`:**
   - `.DS_Store`, `out/`, `bevy-repo/`, `.cargo/`, `.vscode/`, `_temp/`, `*.psd`

3. **Decide: commit or discard the asset experiments.** Figure out which textures/maps the code actually references; the rest are experiments that can go.

4. **Merge `outstanding-changes` into main** — 3 good commits (clippy cleanup, reimplemented improvements, removed unused params). The diff is clean and sensible, though the `physics.rs` changes regress slightly (replaces `if let Some(value)` with `if x.is_some() { x.unwrap() }` — worse style but functional).

5. **Commit `scripts/`** — useful for the project.

6. **Delete stale branches** — all the `MERGED-*`, `mattmp`, `mattmp-explode`, `main-for-explode` branches. Their work has been extracted or is obsolete.
